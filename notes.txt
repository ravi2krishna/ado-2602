    -> Route Tables 

        -> An Azure Route Table is a resource that controls network traffic flow 
        within an Azure Virtual Network (VNet)

        -> You associate a route table with subnets, directing traffic based on destination
        (IP address range) and next hop(Internet, Gateway etc) 

        -> Create a route table when you need to override Azureâ€™s default routing. 
        For example, you can route traffic to a network virtual appliance or to your 
        on-premises network for inspection. 
        A route table contains routes and is associated to one or more subjects.

    -> Considering Our VNET 

        -> We Want Two Types Of Traffic Flow 

            -> Public: Which should be Accessible From Internet 

                -> Web & Application Subnets

            -> Private: Which should not be Accessible From Internet 

                -> Database Subnet 

    -> Network Security Groups - NSG (Firewalls)

        -> Azure network security group is used to "filter" network traffic to and from 
        Azure resources in an Azure virtual network.

        -> A network security group contains "security rules" that "allow" or "deny" 
        inbound network traffic.

        -> For each rule, you can specify source and destination, port, and protocol.

        -> Based On Application Requirements we define rules in NSG 

            -> Myntra Web NSG 

                -> SSH Access (Admin Access For Linux Server) - Port 22 
                -> HTTP Access (For Web Applications) - Port 80 
                -> HTTPS Access (For Web Applications Secured) - Port 443 

            -> Myntra App NSG 

                -> SSH Access (Admin Access For Linux Server) - Port 22 
                -> HTTP Access (For Backends / API) - Port 8080 (8000 - 3000 - 5000 etc)

            -> Myntra DB NSG 

                -> SSH Access (Admin Access For Linux Server) - Port 22 
                -> MySQL DB Access - Port 3306 

            -> NOTE: If using Windows Server, add RDP Access (Admin Access For Windows Server) 
            - Port 3389 


        -> Azure Public IP's 

            -> Public IP addresses allow Internet resources to communicate inbound to 
            Azure resources. Public IP addresses enable Azure resources to communicate to 
            Internet and public-facing Azure services.

            -> For Myntra Web Server & App Server we need an Public IP's as they are 
            internet facing entities.

        -> Azure Private IP's 

            -> Private IP addresses allow Azure resources to communicate within Vnet i.e
            Intranet or local network

            -> NOTE: Every Server will have private ip as primary address and optionally   
            we can add public ip for Internet access

    -> Azure Network Interfaces - NIC 

        -> In Azure, a Network Interface (NIC) is a crucial resource that connects a 
        Virtual Machine (VM) to a Virtual Network (VNet), enabling communication 
        with the internet, other Azure services, and on-premises networks. 
        Each NIC acts as a virtual network adapter, allowing for IP configurations 
        (private/public, static/dynamic), association with Network Security Groups (NSGs) 
        for traffic filtering, and even multiple NICs per VM for advanced connectivity.

        -> NOTE: Azure By default creates Network Interface, when we create Virtual Machine and 
        we cannot attach Custom Network Interface, so we stop server and detach default Network Interface
        and later attach our Custom Network Interface, this is must in GUI   

        -> NOTE: If we Use CLI we can overcome above, while creating Virtual Machine in CLI
        we can attach our custom Network Interface directly

    -> Creating Myntra Infrastructure Using CLI 

        -> az group create --name myntra --location westus2

        -> az network vnet create -g myntra -n myntra-vnet --address-prefix 10.0.0.0/16 

        -> az network vnet subnet create -g myntra --vnet-name myntra-vnet -n myntra-web-sn --address-prefixes 10.0.0.0/24 

        -> az network vnet subnet create -g myntra --vnet-name myntra-vnet -n myntra-app-sn --address-prefixes 10.0.1.0/24 

        -> az network vnet subnet create -g myntra --vnet-name myntra-vnet -n myntra-db-sn --address-prefixes 10.0.2.0/24 

        -> az network route-table create -g myntra -n myntra-pub-rt 

        -> az network route-table route create -g myntra --route-table-name myntra-pub-rt -n www-route --next-hop-type Internet --address-prefix 0.0.0.0/0

        -> az network vnet subnet update --resource-group myntra --vnet-name myntra-vnet --name myntra-web-sn --route-table myntra-pub-rt

        -> az network vnet subnet update --resource-group myntra --vnet-name myntra-vnet --name myntra-app-sn --route-table myntra-pub-rt

        -> az network route-table create -g myntra -n myntra-pvt-rt 

        -> az network route-table route create -g myntra --route-table-name myntra-pvt-rt -n local-route --next-hop-type VnetLocal --address-prefix 10.0.0.0/16 

        -> az network vnet subnet update --resource-group myntra --vnet-name myntra-vnet --name myntra-db-sn --route-table myntra-pvt-rt

        -> az network nsg create -g myntra -n myntra-web-nsg 

        -> az network nsg rule create -g myntra --nsg-name myntra-web-nsg -n myntra-web --priority 100 --source-address-prefixes Internet --destination-port-ranges 22 80 443 --access Allow --protocol Tcp --description "Allow Internet to Web Server on ports 22, 80, 443."

        -> az network nsg create -g myntra -n myntra-app-nsg 

        -> az network nsg rule create -g myntra --nsg-name myntra-app-nsg -n myntra-app --priority 110 --source-address-prefixes Internet --destination-port-ranges 22 8080 --access Allow --protocol Tcp --description "Allow Internet to App Server on ports 22, 8080."

        -> az network nsg create -g myntra -n myntra-db-nsg 

        -> az network nsg rule create -g myntra --nsg-name myntra-db-nsg -n myntra-db-ssh --priority 120 --source-address-prefixes 10.0.0.99/32 --destination-port-ranges 22 --access Allow --protocol Tcp --description "Allow SSH to DB Server on ports 22 from 10.0.0.99"

        -> az network nsg rule create -g myntra --nsg-name myntra-db-nsg -n myntra-db-mysql --priority 130 --source-address-prefixes 10.0.1.0/24 --destination-port-ranges 3306 --access Allow --protocol Tcp --description "Allow MYSQL to DB Server on ports 3306 from all app servers"

        -> az network public-ip create -g myntra -n myntra-web-pip

    -> Azure ARM Templates

        file that defines the infrastructure and configuration for your Azure project 
        using a declarative syntax. It allows you to provision, manage, and deploy

        ensuring consistency and repeatability across different environments.
